● Le problème est clairement hardware, pas logiciel. Quelques suggestions pour stabiliser le RTL-SDR:

  Solutions immédiates:

  1. Hub USB alimenté
    - Le RTL-SDR consomme ~300mA, surtout avec gain élevé
    - Un hub USB avec alimentation externe résoudrait le problème
  2. Réduire le gain dans test_rtlsdr.py et scan406_iq.py
    - Actuellement: set_gain(40) + set_if_gain(20) + set_bb_gain(20) = consommation max
    - Pour signaux faibles à 80km, essayer:
  self.rtl_source.set_gain_mode(True, 0)  # AGC automatique
  # OU gain manuel plus bas:
  self.rtl_source.set_gain(30)
  self.rtl_source.set_if_gain(15)
  self.rtl_source.set_bb_gain(15)
  3. Vérifier l'alimentation
  dmesg | grep -i usb | tail -20  # Voir si erreurs "device not accepting address"
  lsusb -v | grep -A5 "RTL"       # Vérifier si le dongle est détecté
  4. LNA externe (si possible)
    - Amplificateur faible bruit entre l'antenne et le RTL-SDR
    - Compense les signaux faibles à 80km
